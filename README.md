Task 1 — Парсер парковок Алматы (2ГИС)
Собирает структурированные данные по парковкам Алматы через 2ГИС Catalog API и Geocoder API, очищает и выгружает результат для импорта в Google Sheets.
Результат
572 уникальных парковки. Поля: id, название, номер зоны, район, микрорайон, координаты, платная/бесплатная, количество мест (включая спецместа для инвалидов), часы работы, ссылка на карточку 2ГИС.
Готовый файл: `https://docs.google.com/spreadsheets/d/1opdYaXPmT7dJl-ZpkRxi-KtckJOfCUlpYAHW7iCk0dM/edit?usp=sharing`
Зависимости
```bash
pip install requests --break-system-packages
```
Ключ API
Нужен бесплатный demo-ключ 2ГИС Catalog API (получить на dev.2gis.com). Лимиты demo-ключа: 1000 запросов/месяц на Places API и отдельно 1000/месяц на Geocoder API.
Запуск (полный пайплайн)
```bash
# 1. Собрать парковки грубой сеткой по всему городу (\\\~100 запросов)
python parking\\\_scraper.py YOUR\\\_API\\\_KEY
# -> parkings\\\_almaty.csv

# 2. Доуточнить плотные центральные зоны рекурсивным разбиением (\\\~130 запросов)
python parking\\\_scraper.py YOUR\\\_API\\\_KEY --refine
# -> parkings\\\_almaty\\\_refined.csv

# 3. Добавить район/микрорайон через reverse-geocoding (\\\~570 запросов, отдельная квота)
python geocode\\\_parkings.py YOUR\\\_API\\\_KEY parkings\\\_almaty\\\_refined.csv parkings\\\_almaty\\\_geocoded.csv

# 4. Финальная очистка: убрать избыточные колонки, извлечь номер зоны, отсортировать
python clean\\\_dataset.py parkings\\\_almaty\\\_geocoded.csv parkings\\\_almaty\\\_clean.csv
```
Результат — `parkings\\\_almaty\\\_clean.csv`, готов к импорту в Google Sheets (File → Import → Upload).
Подход и обоснование (кратко, подробнее — в SOLUTION_NOTES.md)
Выбрано официальное 2ГИС Catalog API вместо парсинга HTML-страниц: структурированные данные из коробки (capacity, is_paid, schedule), не зависит от вёрстки сайта, легально по ToS. Цена — жёсткий лимит 1000 запросов/месяц на demo-ключе, что потребовало адаптивной сетки запросов (грубое покрытие + рекурсивное доуточнение плотных зон) вместо наивного полного скана.
Известные ограничения
Точный адрес (улица/дом) не реализован — Catalog API для слоя `type=parking` не возвращает `address`; вместо этого используется административная привязка (район/микрорайон) через Geocoder API.
Тариф не реализован — поле `comment`/`address\\\_comment` у объектов парковки в API пустое (проверено на выборке); для получения требовался бы парсинг HTML-карточек, не делали из-за ограничения по времени.
Тип объекта (городская/БЦ/ТЦ/частная) и привязка "к какому объекту относится" не реализованы — у проверенных объектов в наборе все `subtype=ground` (наземные); привязка к ближайшему зданию через доп. запросы технически возможна, но даёт ненадёжную эвристику (решили не делать, см. SOLUTION_NOTES.md).
Покрытие города не стопроцентное: сетка с шагом ~2.6км и углублённой детализацией только в выявленных "горячих точках" центра; окраины с низкой плотностью покрыты грубее.

Task 2 — Рассыльщик WhatsApp (демонстрация принципа)
Программа читает таблицу (номер + текст сообщения) и либо логирует, что было бы отправлено (dry-run, по умолчанию), либо реально отправляет 1–2 сообщения на тестовый/собственный номер (live).
⚠️ Важно про безопасность
dry-run — режим по умолчанию. Без флага `--live` программа никогда не делает сетевых вызовов к WhatsApp.
live ограничен максимум 2 получателями, и только из отдельного файла `live\_whitelist.csv` (не из основной таблицы!). Это жёстко проверяется в коде — даже ошибка во входных данных не приведёт к массовой рассылке.
Номера в `demo\_input.csv` (572 шт.) — синтетические плейсхолдеры (`+7700000001`, `+7700000002`, ...), сгенерированные из данных задачи 1 для демонстрации обработки полного объёма. Это не реальные номера.
Зависимости
```bash
pip install requests --break-system-packages
pip install pywhatkit --break-system-packages   # только для live-режима
```
Для live-режима дополнительно нужно быть залогиненным в WhatsApp Web в браузере на этой машине (отсканировать QR заранее на web.whatsapp.com).
Запуск
Dry-run (по умолчанию, безопасно, без сети)
```bash
python whatsapp\_sender.py --input demo\_input.csv
```
Выведет отчёт `dry\_run\_report.csv` со статусом по каждой строке (ok / invalid_number) и лог в `whatsapp\_sender.log`.
Файл `demo\_input.csv` уже сгенерирован из данных парковок задачи 1. Чтобы пересоздать:
```bash
python generate\_demo\_data.py ../task1/sample\_output.csv demo\_input.csv
```
Live (реальная отправка, максимум 2 номера)
```bash
copy live\_whitelist.example.csv live\_whitelist.csv
# откройте live\_whitelist.csv и впишите свой реальный номер с кодом страны, например +77011234567

python whatsapp\_sender.py --input demo\_input.csv --live --whitelist live\_whitelist.csv
```
Откроется WhatsApp Web в браузере, сообщение отправится автоматически. Результат — `live\_report.csv` + лог.
Обработка ошибок
Проверено на граничных случаях: невалидный формат номера (без `+`, буквы вместо цифр), пустой номер, пустое сообщение, отсутствующий/слишком большой whitelist-файл для live. Во всех случаях программа не падает, пишет понятную причину и продолжает обработку остальных строк (где это применимо).
Известные ограничения
Live-отправка использует автоматизацию WhatsApp Web через `pywhatkit` (открытие браузерной вкладки), а не официальный WhatsApp Business Cloud API — выбрано из-за отсутствия необходимости в Meta Business verification, что не вписалось бы в 5-дневный срок. Подробнее — в SOLUTION_NOTES.md.
Обработка реального обрыва сети в live-режиме оборачивает вызов в `try/except`, но не тестировалась намеренным отключением интернета во время отправки.
